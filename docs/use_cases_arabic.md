# Use Cases - Dawrlk

## UC1: Search Service
Actor: User
Description: البحث عن خدمة محلية.
Flow:
1. فتح التطبيق
2. البحث باسم الخدمة أو اختيار التصنيف
3. عرض المصالح القريبة

## UC2: Book Slot
Actor: User
Description: حجز دور ودفع 20%.
Flow:
1. اختيار المصلحة
2. تحديد الوقت
3. الدفع
4. استلام تأكيد وتذكيرات

## UC3: Notifications
Actor: System
Description: إرسال إشعارات تذكير بالدور
Flow:
1. إرسال تأكيد بعد الحجز
2. تذكير يوم قبل الموعد
3. تذكير ساعتين قبل الموعد
4. تذكير 10 دقائق قبل الموعد

## UC4: Manage Available Slots (Owner)

**Actor:** Business Owner  
**Description:** صاحب المصلحة يمكنه إدارة الأيام والساعات المتوفرة لتقديم خدماته على التطبيق، بحيث يمكن للمستخدمين حجز دور بسهولة.  

### Preconditions:
- المالك مسجل في التطبيق ومصادق عليه  
- المالك دخل للوحة التحكم الخاصة به  

### Flow:
1. المالك يفتح لوحة التحكم الخاصة بالمصلحة  
2. يختار قسم "إدارة الأدوار / الأوقات المتوفرة"  
3. يعرض التطبيق التقويم الأسبوعي أو الشهري  
4. المالك يضيف الأيام التي يقدم فيها الخدمة  
5. يحدد الساعات المتاحة لكل يوم  
6. يحدد نوعية الخدمات لكل يوم/ساعة إذا تطلب الأمر  
7. التطبيق يحفظ التغييرات ويحدث المواعيد المتاحة في النظام  
8. المستخدمين الآن يمكنهم رؤية الأوقات المتاحة وحجزها  

### Postconditions:
- كل المواعيد المتاحة تظهر للمستخدمين تلقائيًا  
- أي حجز مستقبلي يتحقق من الأوقات التي حددها المالك  

### Alternate Flow:
- إذا حاول المالك إضافة وقت متداخل مع وقت موجود → التطبيق يعرض تحذير ويطلب تعديل الوقت  
- إذا لم يقم بحفظ التغييرات → التحديثات لن تنعكس في النظام
